100%

Less "Magic"
  • Configuration by code
  • separated from your application code
    no dependency points to silk*
  • no further runtime dependencies
  • No XML
  • No AOP
  • No annotations
    easy to add but why would you want to ?
  • No accessible mutable state
  • No changing configuration

if-less Modularity

Robust
  • consider in isolation
  • bundles always just installed once
    multiple installs or circles are allowed
  • uninstall bundles
  • configuration through editions, features and constant properties
  • safe renaming
  • typesafe
    seldom just raw-type safety
  • excellent Unit-testability
  • pushes for immutability
    enforces constructor injection
  • data driven
    core uses mostly Value Objects

Features
  • declarative dependency descriptions
    using a guice-like builder
  • multi-binds
  • full generic support
  • easy to extend
    e.g. with Set or List support
  • fast bootsrap
  • about 120 KB jar archive
